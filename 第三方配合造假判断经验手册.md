# 第三方配合造假判断经验手册

## 一、核心定义

### 1.1 什么是"第三方"？

第三方是指上市公司**本身以外**的独立法人或自然人，且**不包括**：
- 上市公司自身（及其全资/控股子公司）
- 上市公司的实际控制人/控股股东（内部决策者，非外部配合方）

### 1.2 什么是"配合造假"？

第三方行为必须构成**"主动配合"**，即明知上市公司实施舞弊，仍提供实质性协助。核心特征：
- **主观故意**：知道或应当知道上市公司在造假
- **实质协助**：提供了造假得以完成的关键环节
- **主动参与**：不是被动配合监管调查，而是主动参与造假链条

## 二、第三方类型判断指南

### 2.1 客户（买方/采购方）

**典型配合行为**：
- 签订无真实商业背景的销售合同
- 提供虚假的验收确认单/收货确认
- 配合开具虚假发票
- 在无实物流转的情况下确认采购

**判断要点**：
- 是否有真实的商品/服务交付？
- 交易价格是否合理？
- 资金是否回流？

**文本关键词**：客户、买方、采购方、下游、销售对象

### 2.2 供应商（卖方/供货方）

**典型配合行为**：
- 签订虚假采购合同
- 提供虚假的发货单据
- 收到货款后资金回流给上市公司
- 配合虚增存货/成本

**判断要点**：
- 是否有真实的货物供应？
- 采购价格是否合理？
- 资金流向是否正常？

**文本关键词**：供应商、供货方、上游、采购对象

### 2.3 银行/金融机构

**典型配合行为**：
- 提供虚假的银行流水/存款证明
- 配合资金循环（出账-回流）
- 违规提供贷款/担保
- 协助隐瞒资金占用

**判断要点**：
- 银行流水是否真实？
- 资金是否在多个账户间循环？

**文本关键词**：银行、支行、分行、金融机构、信托、基金

### 2.4 券商/保荐机构

**典型配合行为**：
- 在IPO/再融资中帮助隐瞒重大事项
- 出具虚假的保荐意见
- 协助编造发行文件

**注意**：仅出具审计/保荐意见不等于配合造假，需有证据表明其**主动参与**造假。

**文本关键词**：券商、保荐、承销、投行

### 2.5 会计师事务所

**典型配合行为**：
- 明知虚假仍出具无保留意见
- 协助调整账目掩盖造假
- 与上市公司合谋修改审计程序

**注意**：仅"审计失败"或"未勤勉尽责"**不等于**配合造假。配合造假需要证据表明事务所**主动参与**。

**文本关键词**：会计师事务所、审计机构、签字注册会计师

### 2.6 评估机构

**典型配合行为**：
- 出具虚假的资产评估报告
- 配合虚增资产价值
- 在明知数据不实的情况下仍出具报告

**文本关键词**：评估公司、资产评估、估值

### 2.7 自然人

**典型配合行为**：
- 代持股份掩盖关联关系
- 提供个人账户配合资金循环
- 作为虚假交易的对手方

**文本关键词**：自然人、个人、代持

### 2.8 其他企业（通道公司/壳公司）

**典型配合行为**：
- 作为资金通道配合资金循环
- 提供壳公司用于虚构交易
- 配合签订无商业实质的合同

**判断要点**：这类公司往往是：
- 无实际经营业务
- 与上市公司或实控人有关联关系（但未披露）
- 主要功能是配合造假

---

## 三、不属于第三方配合的情形

### 3.1 正常业务往来
- 供应商/客户仅作为正常交易对手出现
- 无证据表明其知道或参与造假

### 3.2 审计/评估失败
- 会计师事务所仅"未勤勉尽责"
- 评估机构仅"评估程序不到位"
- 没有证据表明其主动参与造假

### 3.3 被动配合调查
- 在监管调查时提供资料
- 配合监管部门的检查

### 3.4 关联方/控股股东
- 实际控制人、控股股东不属于"第三方"（内部人）
- 子公司不属于"第三方"（合并报表范围内）

### 3.5 仅提及而无配合行为
- 公司名称仅在文本中出现
- 无具体的配合造假行为描述

---

## 四、判断流程

```
Step 1: 文本中是否提到具体的第三方主体？
  ├─ 否 → third_party_flag = 0
  └─ 是 → Step 2

Step 2: 该主体是否属于"第三方"？
  ├─ 上市公司自身/子公司 → 排除
  ├─ 实控人/控股股东 → 排除
  └─ 其他 → Step 3

Step 3: 该主体是否有"主动配合"行为？
  ├─ 仅正常业务往来 → 排除
  ├─ 仅审计/评估失败（无主动参与）→ 排除
  ├─ 签订虚假合同/提供虚假单据 → 纳入
  ├─ 配合资金循环/回流 → 纳入
  ├─ 出借账户/代持股份 → 纳入
  └─ 提供壳公司/通道 → 纳入

Step 4: 提取第三方信息
  ├─ name: 完整法定名称
  ├─ type: 客户/供应商/银行/券商/会计所/评估机构/自然人/其他企业
  └─ role: 具体配合行为描述（50字以内）
```

## 五、类型推断规则

根据文本上下文推断第三方类型：

| 关键词 | 类型 |
|--------|------|
| 客户、买方、采购方、下游 | 客户 |
| 供应商、供货方、上游、采购 | 供应商 |
| 银行、支行、分行、信托、基金 | 银行/金融机构 |
| 券商、保荐、承销、投行 | 券商/保荐机构 |
| 会计师事务所、审计 | 会计师事务所 |
| 评估、资产评估、估值 | 评估机构 |
| 自然人、个人、代持 | 自然人 |
| 控股股东、实控人、关联方 | **排除（非第三方）** |
| 子公司、全资子公司 | **排除（非第三方）** |

## 六、常见误判场景

### 误判1：将控股股东/关联方当作第三方
- 错误：控股股东当代集团 → type=其他企业
- 正确：**排除**（控股股东不属于第三方）

### 误判2：将审计失败当作配合造假
- 错误：会计师事务所未勤勉尽责 → third_party_flag=1
- 正确：**排除**（仅审计失败不等于配合造假）

### 误判3：将名称提取错误导致的分类错误
- 错误：name="评估明显不到位实达集团" → type=评估机构
- 正确：清理名称，正确分类

### 误判4：将子公司/标的公司当作第三方
- 错误：苏州智诚光学科技有限公司（被收购标的）→ third_party_flag=1
- 正确：**排除**（标的公司不属于第三方）

## 七、修复计划

### 问题诊断
当前 STK_labeled 文件中 third_party_flag=1 的 29 行存在以下问题：
1. **类型分布失衡**：35/36 为"其他企业"，无"客户""供应商"
2. **名称提取错误**：部分名称包含垃圾文本（如"评估明显不到位实达集团"）
3. **误判**：控股股东/关联方被当作第三方
4. **类型推断缺失**：`get_third_party_type()` 函数过于简单

### 修复方案

**步骤1：改进类型推断函数 `get_third_party_type()`**
- 基于 Activity 上下文判断类型（而非仅看名称）
- 检查"签订销售合同""采购""客户""供应商"等上下文关键词
- 排除控股股东/关联方/子公司

**步骤2：改进名称清理函数 `clean_third_party_list()`**
- 移除名称中的垃圾前缀（"评估明显不到位""管理层怠于履行向"等）
- 移除重复条目（如同一行出现3次"控股股东当代集团"）

**步骤3：重新运行标注+修正**
- 运行改进后的 annotate_v2.py
- 运行 corrections.py
- 重建 STK_labeled 文件

**步骤4：验证**
- 检查类型分布是否合理
- 抽样检查 10 行第三方标注质量

### 关键文件
- `全量数据标注/annotate_v2.py`：改进 `get_third_party_type()` 和 `identify_third_party()`
- `全量数据标注/corrections.py`：修正误判行
